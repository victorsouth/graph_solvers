# Формат JSON для хранения результатов расчета по слоям

Основная задумка расчетных моделей graph_solvers состоит в задействовании информации о переменных свойствах партий при гидравлическом расчете. Но при это концепция реализации подразумевает большой акцент на декомпозицию гидравлического расчета и транспортного расчета.

Следует отметить, что при транспортном расчете большинство сосредоточенных объектов (lumped) рассчитываются единообразно - эндогенные свойства просто мгновенно передаются со входа на выход. Поэтому выделены две "популяции" моделей - транспортные и гидравлические, причем номенклатура гидравлических моделей шире.

Классы транспортных моделей: source, lumped, pipe
Возможно, будут реализованы специфические транспортные объекты: pump, heater, improver

Классы гидравлических моделей:
source, pipe, pump, valve, heater, improver и другие.

При формировании расчетных результатов по объектам используется концепция разделения параметров на общие и специфические. Общие для всех объектов параметры, такие как расходы, давления, плотности, температуры, хранятся в буферах моделей (буферные результаты). Специфические параметры, такие как ток нагрузки насоса, расход вводимой присадки, зависят от типа объекта.

Учитывая наличие двух популяций моделей нужно уточнить, что поскольку буферы для объектов обоих типов одни и те же, результаты по буферам тоже будут общие. Поскольку популяции гидравлических и транспортных моделей разные, то специфические результаты будут отдельными. Таким образом, имеет три вида результатов: буферные, специфические гидравлические, специфические транспортные

Представлен формат для пакетного (многослойного) расчета. Если нужно выдать результат по одному слою, то массив "layers" будет содержат один элемент.

## Формат JSON для хранения буферных и специфических результатов расчета 

### Буферные параметры по притокам и сосредоченным объектам
- Для сосредоточенных объектов (sources, lumpeds) параметры записываются как одиночные значения, а не массивы
- Для сосредоточенных объектов (lumpeds) параметры записываются в двух объектах: `parameters_in` (вход) и `parameters_out` (выход)

### Буферные параметры для труб
При записи буферных результатов по трубам кроме самих параметров записываются значения достоверности как массивы значений и массивы  с суффиксом `_confidence`

Все параметры профилей зависят от используемой трубы. Помимо параметров вязкости и плотности могут записываться:
- `temperature` / `temperature_confidence` - температура и её достоверность (массивы, в `profiles`)
- `sulfur` / `sulfur_confidence` - содержание серы и её достоверность (массивы, в `profiles`)
- `improver` / `improver_confidence` - концентрация присадки и ее достоверность (массивы, в `profiles`)
- `viscosity0` / `viscosity0_confidence` - вязкость при 0°C и её достоверность (массивы, в `profiles`)
- `viscosity20` / `viscosity20_confidence` - вязкость при 20°C и её достоверность (массивы, в `profiles`)
- `viscosity50` / `viscosity50_confidence` - вязкость при 50°C и её достоверность (массивы, в `profiles`)

### Расшифровка специфиеческих параметров узла ввода присадки (`improvers[].transport`)
- `flow_improver_volumetric_flow20` (строка) - объемный расход присадки при 20°C
- `flow_improver_mass_flow` (строка) - массовый расход присадки
- `concentration` (строка) - концентрация
- `flow_improver_mass_concentration` (строка) - массовая концентрация присадки
- `flow_improver_volumetric_concentration` (строка) - объемная концентрация присадки
- `mode` (строка) - режим работы
- `is_mass_concentraion` (строка) - флаг использования массовой концентрации


### Общие замечания
- Все числовые значения записываются как строки (особенность `boost::property_tree`)
- Числовые строки могут быть записаны в экспоненциальной нотации (например, `"1e-5"` для очень малых или больших чисел)
